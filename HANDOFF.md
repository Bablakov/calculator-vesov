## 1. Задача (что хотели)

Пользователь (тренер/атлет по пауэрлифтингу) вводит свой **одноповторный максимум (1ПМ)** для приседа, жима лёжа и становой тяги. Приложение выводит **процентовку** — рабочие веса по процентам тремя таблицами. Требования по ходу работы:

- Лёгкое, бесплатно раздаётся всем (Android, iOS, ПК), желательно ссылкой/файлом.
- Проценты: **40, 50, 60, 65, 70, 75, 80, 85, 90, 95, 100** (105% = проходка = соревнования, в программах не используется).
- Округление **до 0,1 кг** (НЕ до блинов).
- Сумма троеборья (1ПМ) и **целевая сумма на 105%** (цель цикла) — только как числа.
- История введённых весов.
- Кнопка установки на главный экран (Android + iOS).
- Авто-обновление версии на телефоне.
- QR-код для раздачи в зале, по центру.
- Яркая иконка.

## 2. Ключевые решения и почему (это главное)

| Решение | Почему |
|---|---|
| **PWA (статичный сайт), а не Unity/нативка/Excel/PDF** | Пользователь — Unity-разработчик, но для этой задачи Unity = оверкилл. PWA весит килобайты, раздаётся ссылкой/QR, работает на всех платформах, ставится на экран, офлайн, бесплатный хостинг. |
| **Округление до 0,1 кг, а не до 2,5 кг (блинов)** | Сознательное требование: куда округлять решает тренер/атлет. Показываем точное значение. |
| **ES-модули (js/*.js) + css/styles.css** | Рефакторинг из монолитного index.html. Каждый модуль — своя зона ответственности. |
| **GitHub Pages, деплой = `git push`** | Бесплатно, https из коробки (нужен для PWA/установки), мгновенная раздача по ссылке. |
| **Навигация «сеть-первая» + авто-обновление SW** | Чтобы у людей не зависала старая версия. |
| **105% убрана из программ тренировок** | Проходка = соревнования, не тренировка. В программе максимум 100%. Цель 105% остаётся только как числовая плитка в калькуляторе. |

## 3. Что сделано (на 21.06.2026)

- Ввод 1ПМ для трёх упражнений (кг; принимает запятую/точку, чистит мусор).
- Калькулятор: 4 таблицы недель Н1–Н4 (без 105%), живой пересчёт весов от 1ПМ.
- **Сумма 1ПМ** (троеборье) и **Цель цикла · 105%** — две сводные плитки.
- Округление до 0,1 кг.
- **История**: снимок текущих весов с датой/временем, кнопки «Загрузить» и удалить; хранится локально (лимит 50).
- **Установка на главный экран**: Android — нативный промпт, iOS — инструкция.
- **Поделиться**: QR-код по центру, ссылка, «Скопировать ссылку», системный `navigator.share`.
- **Офлайн** (service worker) и **авто-обновление**.
- PWA-иконки (192/512/180 + SVG), `manifest`, `.nojekyll`.
- Навигация: 5 разделов — Калькулятор · Программы · Журнал · Подсобки · Профиль (по центру).
- **Программы**: «Мои программы» + мастер создания (уровень → дисциплина → упор) → готовая программа от 1ПМ; конструктор (недели → дни → упражнения ≤100% + подсобки к движениям). Старые программы мигрируют автоматически.
- **Журнал**: ПЛАН + ФАКТ (дневник) + история — навигация по **неделям и дням**; в факте — вес+«на снарядах», повторы, качество 1–5, ошибка, заметка.
- **Подсобки**: каталог по категориям + «проблема→подсобки» + добавить своё.
- Раздел «Профиль»: аватар, ФИО/возраст, вес тела с историей, тайминг тренировки, роль Тренер/Атлет.
- Тесты: Node ESM, `tests/`, 79 проверок (`npm test`).

## 6. Как работать дальше

**Локальный запуск** (нужен http для SW):
```bash
python -m http.server 8000
# → http://localhost:8000  (после правок — Ctrl+Shift+R)
```

**Деплой:** `git push` в `main` → Pages пересоберётся за ~минуту.

**Важно при любом изменении файлов:** поднимать версию `CACHE` в `sw.js` (`pct-v9` → `pct-v10` …).

**Тесты:** `npm test`

**Перегенерация ассетов:** `npm run icons` (иконки), `npm install && npm run qr` (QR).
