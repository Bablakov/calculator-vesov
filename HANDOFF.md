## 1. Задача (что хотели)

Пользователь (тренер/атлет по пауэрлифтингу) вводит свой **одноповторный максимум (1ПМ)** для приседа, жима лёжа и становой тяги. Приложение выводит **процентовку** — рабочие веса по процентам тремя таблицами. Требования по ходу работы:

- Лёгкое, бесплатно раздаётся всем (Android, iOS, ПК), желательно ссылкой/файлом.
- Проценты: **40, 50, 60, 65, 70, 75, 80, 85, 90, 95, 100, 105**.
- Округление **до 0,1 кг** (НЕ до блинов).
- Сумма троеборья (1ПМ) и **целевая сумма на 105%** (цель цикла).
- История введённых весов.
- Кнопка установки на главный экран (Android + iOS).
- Авто-обновление версии на телефоне.
- QR-код для раздачи в зале, по центру.
- Яркая иконка.

## 2. Ключевые решения и почему (это главное)

| Решение | Почему |
|---|---|
| **PWA (статичный сайт), а не Unity/нативка/Excel/PDF** | Пользователь — Unity-разработчик, но для этой задачи Unity = оверкилл (25–60 МБ ради формы и таблиц). PWA весит килобайты, раздаётся ссылкой/QR, работает на всех платформах, ставится на экран, офлайн, бесплатный хостинг. Excel/PDF — неудобный ввод на телефоне и плохо расширяется. |
| **Округление до 0,1 кг, а не до 2,5 кг (блинов)** | Сознательное требование пользователя: куда округлять (вверх/вниз) решает тренер или атлет в зависимости от сложности подхода на проходке. Поэтому показываем точное значение до десятых. |
| **Весь код в одном `index.html`** (разметка+CSS+JS inline) | Максимальная простота: один файл «просто работает», легко хостить и шарить. Никакого сборщика. |
| **GitHub Pages, деплой = `git push`** | Бесплатно, https из коробки (нужен для PWA/установки), мгновенная раздача по ссылке. |
| **Навигация «сеть-первая» + авто-обновление SW** | Чтобы у людей не зависала старая версия. Онлайн всегда отдаётся свежий HTML; SW проверяет обновление и тихо перезагружает страницу. Данные в `localStorage` не теряются. |
| **QR — статический `qr.svg`, сгенерён локально (`qrcode`)** | Работает офлайн, без CDN и внешних сервисов. Библиотека только dev-инструмент, в рантайм не попадает. |
| **3 поля и 3 таблицы всегда в один ряд (`repeat(3,1fr)`), размеры на `clamp()`** | Запрос пользователя: таблицы в одну строку на всю ширину. `clamp()` сам ужимает под телефон и растит на десктопе. |
| **Подсветка: 100% — цвет упражнения, 105% — золотой, зебра по «нулям»/«пятёркам»** | Удобная визуальная навигация по длинному списку процентов. |
| **Иконка: оранжево-красный градиент + белая штанга** | Запрос «поярче»: яркий контрастный фон вместо тёмного. |

## 3. Что сделано (возможности)

- Ввод 1ПМ для трёх упражнений (кг; принимает запятую/точку, чистит мусор).
- Три таблицы процентовки 40–105% с живым пересчётом.
- **Сумма 1ПМ** (троеборье) и **Цель цикла · 105%** — две сводные плитки.
- Округление до 0,1 кг.
- **История**: снимок текущих весов с датой/временем, кнопки «Загрузить» и удалить; хранится локально (лимит 50).
- **Установка на главный экран**: Android — нативный промпт (`beforeinstallprompt`), iOS — инструкция (Safari не умеет программно).
- **Поделиться**: QR-код по центру, ссылка, «Скопировать ссылку», системный `navigator.share`.
- **Офлайн** (service worker) и **авто-обновление**.
- PWA-иконки (192/512/180 + SVG), `manifest`, `.nojekyll`.

## 4. Как устроено (кратко; подробно — в `DOCS.md`)

**Файлы:** `index.html` (всё приложение), `sw.js` (офлайн+обновление), `manifest.webmanifest`, иконки (`icon.svg`, `icon-192/512/180.png`), `qr.svg`, генераторы `tools/gen-icons.mjs` (чистый Node, рисует штангу и кодирует PNG вручную через `zlib`) и `tools/gen-qr.mjs` (через dev-зависимость `qrcode`), `package.json` (скрипты `icons`/`qr`).

**Настройки — в начале `<script>` в `index.html`:**
```js
const LIFTS = [ {key,name,accent} … ];                 // упражнения и цвета
const PERCENTS = [40,50,60,65,70,75,80,85,90,95,100,105];
const TARGET_PCT = 105;                                 // целевой процент суммы
```
Цвет «цели» (105% и плитка) — переменная `--target` в `:root` (золотой `#f7c948`).

**Хранение (localStorage, бэкенда нет):**
- `pct.maxes.v1` — текущие значения полей.
- `pct.history.v1` — массив снимков `{id, ts, squat, bench, deadlift}`, новые сверху, лимит 50.

**Расчёты:** `вес = 1ПМ × % / 100`; `Цель = сумма округлённых до 0,1 значений 105%` (совпадает со строками 105% в таблицах).

**Service worker (`sw.js`):** `install` кэширует оболочку + `skipWaiting`; `activate` чистит старые кэши + `clients.claim`; `fetch`: навигация — сеть-первая, прочее — кэш-первый. Версия кэша — константа `CACHE` (сейчас **`pct-v7`**).

## 6. Как работать дальше

**Локальный запуск** (нужен http для SW): `python -m http.server 8000` → http://localhost:8000 (при проверке после правок — **Ctrl+Shift+R**).

**Деплой:** `git push` в `main` → Pages пересоберётся за ~минуту.

**Важно при любом изменении файлов:** поднимать версию `CACHE` в `sw.js` (`pct-v7` → `pct-v8` …) — это сигнал обновления, по нему у всех подтянется свежее.

**Перегенерация ассетов:** `npm run icons` (иконки), `npm install && npm run qr` (QR; URL можно передать аргументом).

**Гочи (на что напороться):**
- Иконка на уже установленном PWA обновляется не сразу — система кэширует; для мгновенного результата переустановить через «На экран Домой».
- Установку/офлайн на реальном телефоне можно проверить только по **https** (живой Pages), не по `http://localhost` с другого устройства.
- На iOS кнопка «На экран Домой» показывает инструкцию (это норма, Safari не ставит программно).
