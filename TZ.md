# ТЗ — Доработка «Калькулятор процентовки»

Документ фиксирует согласованный план Фазы 1 доработки приложения.  
Дата: 2026-06-21 · Статус: **согласовано, ожидает уточнений (см. §7)**

---

## 1. Общий контекст

Приложение — PWA для пауэрлифтинга. Текущий стек: чистый HTML/CSS/JS (ES-модули),
хостинг GitHub Pages, данные в `localStorage`.

**Фаза 1** — весь функционал локально, без бэкенда.  
**Фаза 2** — бэкенд Supabase, профили тренер/атлет, синхронизация между устройствами
(реализуется после того как Фаза 1 обкатана; токены/ключи Supabase будут предоставлены позже).

## 2. Навигация — новая структура (5 разделов)

```
┌────────────┬───────────┬──────────┬──────────┬──────────┐
```

## 3. Экран 1 — Калькулятор (переработка)

### 3.1 Что убрать / изменить

- **Убрать строку 105%** из всех таблиц программ и цикла.
- **Оставить** плитку «Цель цикла · 105%» как числовое значение (не строка таблицы).
- Раздел «Цикл» (с переключением недели/дня) заменяется 4 вертикальными таблицами
  без переключения — все 4 недели видны сразу на одном экране.

### 3.2 Макет (сверху вниз)

```
┌──────────────────────────────────────────────────┐
│  [Присед]      [Жим лёжа]     [Становая]         │  ← поля ввода 1ПМ
│  Сумма 1ПМ: 580 кг     Цель цикла · 105%: 609 кг │  ← плитки
├──────────────────────────────────────────────────┤
│  НЕДЕЛЯ 1 · средняя                               │
│  %      Присед    Жим      Тяга                  │
│  65%    130       91       156                   │
│  75%    150       105      180                   │
│  85%    170       119      204                   │
├──────────────────────────────────────────────────┤
│  НЕДЕЛЯ 2 · средняя                               │
│  70% / 80% / 90%  ...                            │
├──────────────────────────────────────────────────┤
│  НЕДЕЛЯ 3 · тяжёлая                               │
│  75% / 85% / 95%  ...                            │
├──────────────────────────────────────────────────┤
│  НЕДЕЛЯ 4 · лёгкая                                │
│  40% / 50% / 60%  ...                            │
└──────────────────────────────────────────────────┘
```

### 3.3 Матрица недель (итоговая, без 105%)

| Неделя | Характер  | Подходы (%)      |
|--------|-----------|-----------------|
| Н1     | средняя   | 65 · 75 · 85    |
| Н2     | средняя   | 70 · 80 · 90    |
| Н3     | тяжёлая   | 75 · 85 · 95    |
| Н4     | лёгкая    | 40 · 50 · 60    |

### 3.4 Типы тренировок (влияют на количество столбцов)

Тип выбирается при создании **программы** (не в калькуляторе глобально):

| Тип          | Упражнения                        |
|--------------|-----------------------------------|
| Троеборье    | Присед + Жим лёжа + Становая тяга |
| Двоеборье    | Жим лёжа + Становая тяга          |
| Одно движение | Только одно (выбор: присед / жим / тяга) |

Калькулятор на первом экране по умолчанию показывает троеборье.
Если у пользователя есть активная программа — берёт тип из неё.

### 3.5 История

Остаётся как есть (снимки 1ПМ с датой, загрузить / удалить).

## 4. Экран 2 — Программы

### 4.1 Структура

```
Мои программы              Шаблоны (только просмотр)
─────────────────          ─────────────────────────
[+ Создать]                [Наш 4-нед. цикл  →]
                           [5/3/1             →]
[Моя прога 1  →]           [Шейко             →]
[Моя прога 2  →]           [Куб               →]
```

### 4.2 Шаблоны (зашиты нами)

- **Наш 4-нед. цикл** — текущая матрица (Н1–Н4, см. §3.3)
- **5/3/1** — классическая программа Джима Вендлера (3 рабочих подхода + АМРАП)
- **Шейко** — базовый блок Шейко (большой объём, % от 1ПМ)
- **Куб** — Cube Method Брэндона Лилли (ротация: взрывная/тяжёлая/объёмная)

Шаблоны: **только просмотр**. Кнопка «Взять за основу» создаёт редактируемую копию
в разделе «Мои программы».

### 4.3 Конструктор программы

При создании/редактировании задаётся:
- Название
- Тип: Троеборье / Двоеборье / Одно движение (+ выбор упражнения)
- Количество недель (от 1 до 12)
- Для каждой недели: название/характер + список подходов (% и повт для каждого упражнения)

**Правило:** 105% (проходка) в конструкторе недоступна — максимум 100%.
Проходка = соревнования, не тренировка.

## 5. Экран 3 — Журнал

Два подраздела (переключение табом):

### 5.1 ПЛАН

- Выбор активной программы
- Навигация по неделям и дням
- Отображение: упражнение → список подходов (% · кг · повт)
- Только просмотр (редактирование — в Конструкторе)

### 5.2 ФАКТ (дневник выполнения)

- Кнопка «Начать тренировку» → открывает текущий день плана
- Для каждого подхода:
  - Факт. повторения (число)
  - Факт. вес, кг (с подсказкой «на снарядах» — кратно 2,5 кг)
  - Качество: 1–5 (Ужасно / С ошибками / Нормально / Хорошо / Отлично)
  - Ошибка: выпадающий список из каталога `ERRORS[liftKey]` (необязательно)
  - Заметка: свободный текст (необязательно)
- Кнопка «Завершить тренировку» → запись в историю

### 5.3 История тренировок

- Список прошлых тренировок (дата, программа, неделя/день, сумма кг)
- Просмотр деталей каждой

## 6. Экран 4 — Подсобки

Два подраздела:

### 6.1 КАТАЛОГ

Упражнения разбиты по категориям:

| Категория      | Примеры                                     |
|---------------|---------------------------------------------|
| Жим / плечи   | Жим узким хватом, Дипы, Разводка, ОХП      |
| Присед / ноги | Фронтальный присед, Жим ногами, Выпады     |
| Тяга / спина  | Тяга в наклоне, Тяга с плинтов, Румынская  |
| Корпус        | Гиперэкстензии, Планка, «Доброе утро»       |
| ОФП           | Подтягивания, Отжимания, Фермерская ходьба |

- Карточка упражнения: название, мышцы, как выполнять, с чем помогает
- Кнопка «+ Добавить своё» — пользователь создаёт кастомное упражнение

### 6.2 ПРОБЛЕМЫ → ПОДСОБКИ

Берём каталог ошибок из `content.js` (ERRORS) и расширяем:

```
Присед:
  ├── Не встал / завалился    → [список подсобок]
  ├── Округлил спину          → [список подсобок]
  ├── Колени ушли внутрь      → [список подсобок]
  ├── Пятки оторвались        → [список подсобок]
  └── Не досел                → [список подсобок]

Жим лёжа:
  ├── Не коснулся груди       → [список подсобок]
  ├── Оторвал таз             → [список подсобок]
  ├── Перекос штанги          → [список подсобок]
  └── Болтанка в локтях       → [список подсобок]

Становая тяга:
  ├── Согнул руки             → [список подсобок]
  ├── Округлил поясницу       → [список подсобок]
  ├── Штанга ушла вперёд      → [список подсобок]
  ├── Не довёл таз            → [список подсобок]
  └── Рывок спиной            → [список подсобок]
```

**Связи «проблема → подсобки» зашиваем мы** (не настраивается пользователем).

## 10. Технические правила (не меняются)

- Стек: чистый HTML/CSS/JS (ES-модули), без фреймворков и сборщиков
- Хранение Фаза 1: `localStorage` (ключи по схеме `pct.*` / `profile.*`)
- Деплой: `git push` в `main` → GitHub Pages
- После любых изменений файлов: поднимать `CACHE` в `sw.js`
- Тесты: `npm test` (Node ESM, папка `tests/`)
- Локальный сервер: `python -m http.server 8000`
