## 1. Разделы и возможности

Навигация — 5 разделов (по центру), переключение по hash.

**Калькулятор.** Ввод 1ПМ (присед/жим/становая) → **4 таблицы недель** (Н1–Н2 средние, Н3 тяжёлая, Н4 лёгкая) сразу на экране; столбцы — по упражнениям (троеборье). Плитки «Сумма 1ПМ» и числовая «Цель цикла · 105 %». **105 % в таблицах нет** (проходка = соревнования, не тренировка). Нет 1ПМ → ячейки «—». История снимков 1ПМ (загрузить/удалить, лимит 50).

**Программы.** «Мои программы» (создать / изменить / удалить) и **шаблоны** (наш 4-нед. цикл, 5/3/1, Шейко, Куб) — только просмотр, кнопка «Взять за основу» создаёт редактируемую копию. **Конструктор:** название; тип (Троеборье / Двоеборье / Одно движение + выбор упражнения); 1–12 недель; для каждой недели — характер и список подходов (`%` ≤ 100 и повторы — число или «макс»).

**Журнал.**
- **ПЛАН** — выбор активной программы, навигация по неделям, веса подходов от текущего 1ПМ (`% · кг · повт`).
- **ФАКТ** — «Начать тренировку» → по каждому подходу: факт. вес (с быстрым выбором кратного 2,5 «на снарядах»), факт. повторы, качество 1–5, типовая ошибка, заметка → «Завершить тренировку».
- **История** — список тренировок (дата, программа, неделя, объём кг), раскрытие деталей, удаление.

**Подсобки.**
- **Каталог** по категориям (Жим/плечи, Присед/ноги, Тяга/спина, Корпус, ОФП): карточки с мышцами, техникой и тем, чему помогают; «Добавить своё».
- **Проблема → подсобки** — типовые ошибки по упражнениям и подобранные под них подсобки (связи зашиты).

**Профиль.** Аватар (загрузка фото), ФИО, возраст; вес тела с историей замеров; тайминг тренировки (начало/конец/длительность, ручная правка); роль Тренер/Атлет (для Фазы 2).

## 3. Модель данных (localStorage)

| Ключ | Что хранит |
|---|---|
| `pct.maxes.v1` | Текущие 1ПМ `{ squat, bench, deadlift }` (общие для всех разделов) |
| `pct.history.v1` | История снимков 1ПМ `[{ id, ts, squat, bench, deadlift }]` |
| `pct.programs.v1` | Мои программы `[{ id, name, type, lift, weeks:[{kind, sets:[{pct,reps}]}] }]` |
| `pct.active.v1` | id активной программы (для журнала) |
| `pct.sessions.v1` | История тренировок `[{ id, ts, programName, week, entries, volume }]` |
| `pct.acc.custom.v1` | Свои подсобки `[{ id, name, cat }]` |
| `profile.info.v1` | `{ first, last, middle, age, avatar, role }` |
| `profile.weight.v1` | История веса тела `[{ id, ts, kg }]` |
| `profile.timing.v1` | `{ start, end, day }` |

## 4. Ключевая логика

**Матрица недель** (`config.js → WEEKS`): на неделю — список подходов `{pct, reps}`; `reps` — число или `"макс"` (АМРАП). По умолчанию: Н1 65/75/85, Н2 70/80/90, Н3 75/85/95, Н4 40/50/60. **105 % не используется** (максимум 100 %).

**Округление** (`util.js`): `round1` — до 0,1 кг (показываем точное значение, направление округления решает атлет). `round2_5` / `plateOptions` — ближайшее кратное 2,5 кг и две ближайшие «стороны» (для выбора фактического веса в журнале).

**План на неделю** (`journal.js → planForWeek`): для каждого упражнения программы (по её типу) и каждого подхода недели — вес `1ПМ × % / 100` (точный + «на снарядах»).

**Объём тренировки** (`journal.js → sessionVolume`): Σ(факт.вес × факт.повторы) по заполненным подходам.

**Типы программ** (`programs.js → liftsForType`): троеборье = присед+жим+тяга; двоеборье = жим+тяга; одно движение = выбранное.

## 5. Настройка под себя

Всё предметное — в `js/config.js`:
- `LIFTS` — упражнения (`key`, `name`, `short`, цвет `accent`).
- `WEEKS` — матрица недель (характер + подходы).
- `SPLIT` — дни сплита (задел).
- `TARGET_PCT` — процент цели в калькуляторе (105).

Шаблоны программ — в `js/templates.js`. Контент ошибок/подсобок — `js/content.js` (`ERRORS`) и `js/accessories.js` (`CATALOG`, `PROBLEM_FIX`).

После любых правок — поднять `CACHE` в `sw.js`.

## 6. Тесты

```bash
npm test     # node tests/run.mjs
```
Покрыто (node ESM, без DOM): матрица и округление (`cycle.test`), профиль (`profile.test`), программы/типы/валидация (`programs.test`), план и объём журнала (`journal.test`), полнота подсобок (`accessories.test`). В `package.json` стоит `"type":"module"`.

Правило проекта: **фича = код + тесты + документация**; баг → сначала падающий регрессионный тест, потом фикс.

```bash
python -m http.server 8000   # → http://localhost:8000
```
SW и установка требуют http(s)/localhost. После правок — **Ctrl+Shift+R**.

## 8. Публикация и обновления

Деплой = `git push` в `main` (Pages: `main` / `root`). Навигация HTML — «сеть в приоритете», прочее — «кэш в приоритете»; новая версия активируется и тихо перезагружает страницу (данные в `localStorage` не теряются).

**При любом изменении файлов поднимай `CACHE` в `sw.js`** (`pct-v11` → `pct-v12` …) — это сигнал обновления.

## 9. Решение проблем

| Симптом | Что сделать |
|---|---|
| Не вижу свежую версию | Свернуть/открыть заново; в браузере — Ctrl+Shift+R |
| После правок у людей старое | Поднять `CACHE` в `sw.js` и запушить |
| Нет установки/офлайна на телефоне | Нужен https — проверять на живом Pages |
| iOS «На экран Домой» молчит | Норма: показывается инструкция (Safari не ставит программно) |
