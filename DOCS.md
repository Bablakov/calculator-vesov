## 1. Разделы и возможности

Навигация — 7 разделов (по центру), переключение по hash: Калькулятор · Программы · Журнал · Отчёты · Подсобки · Профиль · Админ.

**Калькулятор.** Ввод 1ПМ (присед/жим/становая) → **4 таблицы недель** (Н1–Н2 средние, Н3 тяжёлая, Н4 лёгкая) сразу на экране; столбцы — по упражнениям (троеборье). Плитки «Сумма 1ПМ» и числовая «Цель цикла · 105 %». **105 % в таблицах нет** (проходка = соревнования, не тренировка). Нет 1ПМ → ячейки «—». История снимков 1ПМ (загрузить/удалить, лимит 50).

**Программы.** «Мои программы» (изменить / удалить) и **мастер создания** в три шага: **уровень** (Начинающий / Любитель / Профессионал) → **дисциплина** (Троеборье / Двоеборье / Жим / Становая) → **упор цикла** (напр. для тяги: срыв с пола / дожим / скорость). На выходе — готовая программа с весами от 1ПМ; «Взять за основу» открывает её в конструкторе. **Конструктор и превью** листаются по **неделям и дням** (пилюли, как в журнале; по умолчанию 3 дня): название; неделя → дни → упражнения (основное движение: `%` ≤ 100 × повторы) + **подсобки** к каждому движению (подходы × повторы). Контент шаблонов — черновик, правится позже в админ-панели (см. `TZ.md` §11.8).

**Журнал.**
- **ПЛАН** — выбор активной программы, навигация по **неделям и дням**, веса подходов от текущего 1ПМ (`% · кг · повт`) + список подсобок дня.
- **ФАКТ** — **автопрогрессия** (подсказывает «Следующая тренировка #N · неделя · день»). «Начать» → **вес тела** и **тайминг** (начало = время устройства, конец вручную/авто). Упражнения листаются **плашечками и стрелками** (по одному за раз), два блока: **«Основной блок»** (база) и **«Подсобка»** (подсобки тоже логируются). По подходу: вес (быстрый выбор «на снарядах»), повторы, качество 1–5, ошибка/заметка + **таймер подхода** (старт/стоп → время под нагрузкой и отдыха). **Пустые вес/повторы при завершении = план** (качество/заметки — нет). «Завершить» (вес тела уходит и в профиль).
- **История** — список тренировок (дата, программа, неделя/день, объём, длительность, вес тела); раскрытие деталей; **«Изменить»** — открыть прошлую тренировку и отредактировать; удаление.

**Подсобки.**
- **Каталог** по категориям (Жим/плечи, Присед/ноги, Тяга/спина, Корпус, ОФП): карточки с мышцами, техникой и тем, чему помогают; «Добавить своё».
- **Проблема → подсобки** — типовые ошибки по упражнениям и подобранные под них подсобки (связи зашиты).

**Профиль.** Аватар (загрузка фото), ФИО, **дата рождения** (возраст считается автоматически); **вес тела** — «текущий» подставляется из последней тренировки, правится вручную, есть история замеров. (Тайминг переехал в тренировку; роль Тренер/Атлет убрана — вернётся в Фазе 2.)

**Отчёты.** Сводка по завершённым тренировкам: число тренировок, средняя длительность и её диапазон, частота (трен/нед), средний и суммарный объём, среднее качество, **среднее время отдыха и под нагрузкой** (с таймера), тоннаж по движениям. Пусто — пока нет тренировок.

**Админ.** Правка **контента** (логика не меняется): тексты ошибок, связи «проблема → подсобки», каталог подсобок и **базовый шаблон** (вкладка «Шаблон»: матрица недель + базовые подсобки по движениям). Правки сохраняются локально и переопределяют дефолты во всех разделах (матрица/подсобки — в генераторе шаблонов); «Сбросить» возвращает стандарт. Пользователь = админ сайта.

| Ключ | Что хранит |
|---|---|
| `pct.maxes.v1` | Текущие 1ПМ `{ squat, bench, deadlift }` (общие для всех разделов) |
| `pct.history.v1` | История снимков 1ПМ `[{ id, ts, squat, bench, deadlift }]` |
| `pct.programs.v1` | Мои программы `[{ id, name, level, discipline, focus, weeks:[{kind, days:[{name, exercises:[{key, sets:[{pct,reps}], acc:[{name,sets,reps}]}]}]}] }]` |
| `pct.active.v1` | id активной программы (для журнала) |
| `pct.sessions.v1` | История тренировок `[{ id, ts, programName, week, day, bw, start, end, durationMin, workSec, restSec, setsTimed, items:[{kind:"main"|"acc", name, key?, sets:[{w,reps,q,err?,note,st?,et?,plan*}]}], volume }]` (старые записи — `entries`-map, читаются для совместимости) |
| `pct.admin.weeks.v1` / `pct.admin.accdef.v1` | Правка админа: матрица недель / базовые подсобки (нет ключа = дефолт) |
| `pct.acc.custom.v1` | Свои подсобки `[{ id, name, cat }]` |
| `pct.admin.errors.v1` | Правка админа: тексты ошибок (или нет ключа = дефолт) |
| `pct.admin.catalog.v1` | Правка админа: каталог подсобок |
| `pct.admin.fix.v1` | Правка админа: связи «проблема → подсобки» |
| `profile.info.v1` | `{ first, last, middle, birth, avatar }` |
| `profile.weight.v1` | История веса тела `[{ id, ts, kg }]` |
| `profile.timing.v1` | `{ start, end, day }` |

## 4. Ключевая логика

**Матрица недель** (`config.js → WEEKS`): на неделю — список подходов `{pct, reps}`; `reps` — число или `"макс"` (АМРАП). По умолчанию: Н1 65/75/85, Н2 70/80/90, Н3 75/85/95, Н4 40/50/60. **105 % не используется** (максимум 100 %).

**Округление** (`util.js`): `round1` — до 0,1 кг (показываем точное значение, направление округления решает атлет). `round2_5` / `plateOptions` — ближайшее кратное 2,5 кг и две ближайшие «стороны» (для выбора фактического веса в журнале).

**План на день** (`journal.js → planForDay`): для каждого упражнения дня и каждого подхода — вес `1ПМ × % / 100` (точный + «на снарядах»); подсобки прокидываются как есть (подходы × повторы).

**Объём тренировки** (`journal.js → sessionVolume`): Σ(факт.вес × факт.повторы) по заполненным подходам.

**Дисциплины** (`programs.js → liftsForDiscipline`): троеборье = присед+жим+тяга; двоеборье = жим+тяга; жим / становая = одно движение. Генерация программы — `templates.js → buildTemplate(level, discipline, focus)`; старые программы переносятся `programs.js → migrateProgram` (формат `weeks[{sets}]` → `weeks[{days[{exercises}]}]`).

**Автопрогрессия журнала** (`journal.js → flattenSlots`/`nextSlot`): по числу завершённых тренировок программы вычисляет следующую неделю/день (по кругу). **Длительность** — `durationMin(start, end)`.

**Тренировка (ФАКТ)** (`journal.js`): `buildDayItems(program, maxes, w, d)` — разворачивает день в «элементы» (основные движения + подсобки как отдельные упражнения с подходами); `itemsVolume(items)` — тоннаж по факту; `finalizeItems(items)` — подставляет план в пустые вес/повторы; `restWorkStats(items)` — время под нагрузкой/отдыха из меток `st/et` подходов (таймер). Элементы сохраняются в сессию (`items`), история самодостаточна и редактируема.

**Базовый шаблон через админку** (`contentstore.js`): `getWeeks()` / `getAccDefaults()` (правка админа из `localStorage` либо дефолты `config.WEEKS` / `accessorydata.ACC_DEFAULTS`) — их читает `templates.buildTemplate`, поэтому правки в «Админ → Шаблон» меняют генерацию программ. **Отчёты:** `reports.tempoStats(sessions)` — среднее время отдыха/нагрузки.

**Отчёты** (`reports.js`): `durationStats`, `volumeStats`, `weeklyFrequency`, `liftTonnage`, `qualityAvg` — чистые функции над списком сессий.

**Контент с правками** (`contentstore.js`): `getErrors/getCatalog/getProblemFix` возвращают правку админа из `localStorage` либо встроенный дефолт; `save*`/`reset*` вызываются из раздела «Админ». Потребители (журнал, подсобки) читают контент только отсюда.

## 5. Настройка под себя

Всё предметное — в `js/config.js`:
- `LIFTS` — упражнения (`key`, `name`, `short`, цвет `accent`).
- `WEEKS` — матрица недель (характер + подходы).
- `SPLIT` — дни сплита (задел).
- `TARGET_PCT` — процент цели в калькуляторе (105).

Контент программ (уровни/дисциплины/упоры/подсобки) и генератор шаблонов — в `js/templates.js`. Контент ошибок/подсобок — `js/content.js` (`ERRORS`) и `js/accessories.js` (`CATALOG`, `PROBLEM_FIX`). Всё это — черновик под будущую админ-панель (`TZ.md` §11.8).

После любых правок — поднять `CACHE` в `sw.js`.

## 8. Публикация и обновления

Деплой = `git push` в `main` (Pages: `main` / `root`). Навигация HTML — «сеть в приоритете», прочее — «кэш в приоритете»; новая версия активируется и тихо перезагружает страницу (данные в `localStorage` не теряются).

**При любом изменении файлов поднимай `CACHE` в `sw.js`** (`pct-v14` → `pct-v15` …) — это сигнал обновления.

## 9. Решение проблем

| Симптом | Что сделать |
|---|---|
| Не вижу свежую версию | Свернуть/открыть заново; в браузере — Ctrl+Shift+R |

| Нет установки/офлайна на телефоне | Нужен https — проверять на живом Pages |
| iOS «На экран Домой» молчит | Норма: показывается инструкция (Safari не ставит программно) |

## 10. Дорожная карта

- **Фаза 1 / 1.5** — готово: калькулятор по неделям; Программы (мастер уровень→дисциплина→упор, недели→дни→подсобки, конструктор); Журнал (план/факт по неделям и дням, автопрогрессия, вес тела и тайминг в тренировке); Отчёты; Подсобки; Профиль (ФИО, дата рождения, вес тела); Админ-панель контента.
- **Фаза 2** — бэкенд **Supabase**: авторизация, профили Тренер/Атлет (тренер видит атлетов и их журналы), синхронизация между устройствами. Токены/ключи — позже.
