## Обзор

Лёгкое веб-приложение (**PWA**) для пауэрлифтинга. Атлет вводит одноповторный максимум (1ПМ) для приседа, жима лёжа и становой тяги — приложение показывает рабочие веса по процентам (40–105%) тремя таблицами, считает сумму троеборья и целевую сумму на конец цикла.

Почему web/PWA, а не нативное приложение или Unity:

- **Весит килобайты**, открывается мгновенно, не нужен магазин приложений.
- **Раздаётся ссылкой или QR-кодом** — работает одинаково на Android, iOS и ПК.
- **Ставится на главный экран** как приложение и **работает офлайн**.
- **Бесплатный хостинг** на GitHub Pages, обновления — простым `git push`.

## Возможности

- Ввод 1ПМ для трёх упражнений (кг; принимает запятую и точку).
- Три таблицы процентов **40, 50, 60, 65, 70, 75, 80, 85, 90, 95, 100, 105 %**.
- **Сумма 1ПМ** (троеборье) и **цель цикла 105 %** — две сводные плитки.
- Округление весов до **0,1 кг** (направление округления выбирает тренер/атлет).
- **История**: снимки введённых весов с датой; загрузка обратно в поля и удаление.
- **Установка на главный экран** (Android — нативно, iOS — по инструкции).
- **QR-код и ссылка** для раздачи, кнопки «Скопировать» и системное «Поделиться».
- **Офлайн-режим** и **авто-обновление** до свежей версии.
- Цветовая навигация: 100 % — цвет упражнения, 105 % — золотой (цель), зебра по «нулям»/«пятёркам».

## Как пользоваться

1. Открой приложение (сайт или иконку с главного экрана).
2. Введи свой 1ПМ в поля **Присед / Жим лёжа / Становая тяга** (в килограммах).
3. Сразу появятся:
   - три таблицы с весами по процентам;
   - **Сумма 1ПМ** — сумма трёх максимумов;
   - **Цель цикла · 105 %** — к чему стремиться к концу цикла.
4. **💾 Сохранить** — записать текущие веса в *Историю* (с датой и временем).
   В истории: **Загрузить** — вернуть запись в поля, **✕** — удалить.
5. **📲 На экран «Домой»** — установить приложение (см. [установку](#обновления-у-пользователей)).
6. **Сбросить** — очистить поля.
7. Внизу — **QR-код и ссылка**: дай отсканировать или поделись для раздачи в зале.

Введённые веса и история **сохраняются на устройстве** — при следующем открытии всё на месте.

> Веса показаны с точностью до 0,1 кг намеренно: округлять вверх или вниз решает тренер
> или атлет в зависимости от сложности подхода на проходке.

## Настройка под себя

Всё настраивается в начале блока `<script>` в `index.html`:

```js
// Упражнения и их цвета. Чтобы добавить ещё одно — допишите строку.
const LIFTS = [
  { key:"squat",    name:"Присед",        accent:"#ff5d3b" },
  { key:"bench",    name:"Жим лёжа",      accent:"#3b82f6" },
  { key:"deadlift", name:"Становая тяга", accent:"#22c55e" },
];

// Ряд процентов — единый для всех таблиц.
const PERCENTS = [40, 50, 60, 65, 70, 75, 80, 85, 90, 95, 100, 105];

// Процент целевой суммы цикла.
const TARGET_PCT = 105;
```

- **Добавить упражнение** — новая строка в `LIFTS` (уникальный `key`, имя, цвет). Поле ввода и таблица создадутся автоматически.
- **Изменить проценты** — отредактируй массив `PERCENTS` (любые значения и шаг).
- **Сменить целевой процент** — `TARGET_PCT`.
- **Цвет «цели» (105 % и плитка)** — переменная `--target` в `:root` (по умолчанию золотой `#f7c948`).

После правок подними версию кэша в `sw.js` (см. [обновления](#обновления-у-пользователей)).

## Как устроено (архитектура)

### Поток данных и расчёты

- `parseNum(v)` — превращает ввод («82,5», «82.5 кг») в число; мусор и значения ≤ 0 → `0`.
- `round1(kg)` — округление до 0,1 кг.
- `fmt(kg)` — округляет до 0,1 и убирает хвост `.0` («100», «83.8»).
- `recalc()` — на каждый ввод пересчитывает все ячейки (`вес = 1ПМ × % / 100`), а также:
  - **Сумма 1ПМ** = сумма введённых максимумов;
  - **Цель 105 %** = сумма округлённых значений 105 % каждого упражнения (совпадает с тем, что видно в строках 105 % таблиц).

### Хранение (localStorage)

Данные не уходят с устройства, бэкенда нет. Ключи:

| Ключ | Что хранит |
|---|---|
| `pct.maxes.v1` | текущие значения полей `{ squat, bench, deadlift }` |
| `pct.history.v1` | массив снимков `{ id, ts, squat, bench, deadlift }`, новые сверху, лимит 50 |

### Подсветка строк

Класс строки задаёт `rowClass(p)`:

- `100 %` → `is-max` — цвет упражнения (рабочий максимум);
- `105 %` → `is-target` — золотой (цель/над-максимум);
- кратные 10 (`40, 50, 60…`) → `dec0` — чуть светлее;
- оканчивающиеся на 5 (`65, 75, 85…`) → `dec5` — темнее и сероватее.

Так список читается «полосами». Вёрстка: поля и таблицы — по 3 в ряд (`repeat(3,1fr)`),
размеры на `clamp()` (сами ужимаются под телефон и растут на десктопе).

### PWA: офлайн и авто-обновление

`sw.js`:

- **install** — кэширует «оболочку» (`ASSETS`) и сразу активируется (`skipWaiting`).
- **activate** — удаляет старые кэши (по имени `CACHE`) и берёт управление (`clients.claim`).
- **fetch** — навигация (HTML) идёт **«сеть в приоритете»** (онлайн всегда свежая версия, офлайн — из кэша); прочее (иконки, QR, манифест) — **«кэш в приоритете»**.

В `index.html` регистрация SW дополнительно:

- если страница уже под управлением SW (не первый визит) — при активации новой версии **тихо перезагружает** страницу (данные в `localStorage` не теряются);
- вызывает `reg.update()` при запуске и при возврате к приложению (`visibilitychange`).

Итог: пользователю **не нужно ничего чистить** — свежая версия подтягивается сама.

### Иконки и QR (генераторы)

- `tools/gen-icons.mjs` — рисует штангу и кодирует PNG **вручную** (через встроенный `zlib`), без сторонних пакетов. Запуск: `node tools/gen-icons.mjs` (или `npm run icons`).
- `tools/gen-qr.mjs` — делает `qr.svg` библиотекой `qrcode`. Запуск: `npm run qr` (нужен один `npm install`).

## Локальная разработка

Service worker и установка требуют `http(s)` (или `localhost`). Поднимаем статический сервер из корня:

```bash
python -m http.server 8000
# открой http://localhost:8000
```

Сам калькулятор работает и при простом открытии `index.html` двойным кликом — но без офлайна и установки (это даёт только запуск по http).

При проверке после изменений жми **Ctrl + Shift + R** (жёсткая перезагрузка), чтобы обойти кэш.

## Публикация на GitHub Pages

Деплой = `git push` в ветку `main`. Pages настроен на источник **`main` / `root`** и пересобирается сам за ~минуту.

Первичная настройка (уже сделана): **Settings → Pages → Source: Deploy from a branch → `main` / `/ (root)`**.

```bash
# обычный цикл обновления
git add -A
git commit -m "..."
git push           # через ~минуту изменения на https://bablakov.github.io/calculator-vesov/
```

## Обновления у пользователей

**Главное: пользователю ничего делать не нужно — приложение обновляется само.**

Как это работает:

1. Код приложения лежит в `index.html`, который отдаётся «сеть в приоритете» — при заходе **онлайн** всегда подтягивается свежая версия.
2. Service worker проверяет обновление при запуске и при возврате к приложению; найдя новую версию, активирует её и **тихо перезагружает** страницу.
3. Введённые веса и история в `localStorage` — перезагрузка проходит незаметно.

**Сигнал «вышла новая версия» — версия `CACHE` в `sw.js`.** При любом изменении файлов
поднимай её (`pct-v5` → `pct-v6` …): старый кэш очистится, у всех подтянется свежее.

Единственное исключение — **полный офлайн**: показывается последняя сохранённая версия,
пока не появится интернет (это и есть смысл офлайн-режима).

### Установка на главный экран

- **Android (Chrome):** кнопка «📲 На экран Домой» вызывает нативную установку; либо меню `⋮` → «Установить приложение».
- **iOS (Safari):** кнопка покажет инструкцию: «Поделиться» → «На экран Домой» → «Добавить» (Safari не умеет ставить программно).

## Решение проблем

| Симптом | Что сделать |
|---|---|
| Не вижу свежую версию | Свернуть/открыть приложение заново; в браузере — **Ctrl + Shift + R** |
| Совсем «застряло» | Удалить иконку с экрана и поставить заново через «На экран Домой» |
| QR не сканируется | Увеличить яркость экрана; QR — тёмный на белом, держать ровно |
| Кнопка «На экран Домой» молчит на iOS | Это нормально: на iOS она показывает инструкцию, установка — вручную через «Поделиться» |
| Не ставится/нет офлайна на телефоне | Нужен **https** — проверять на живом адресе Pages, не на `http://localhost` с другого устройства |
| После правок у людей старое | Поднять версию `CACHE` в `sw.js` и запушить |
