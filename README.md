# Калькулятор процентовки

Лёгкое веб-приложение (PWA): вводишь одноповторный максимум (присед, жим лёжа, становая тяга) — получаешь рабочие веса от 40% до 105% тремя таблицами. Веса округляются до 0,1 кг (направление округления выбирает тренер/атлет).

Работает на Android, iOS и ПК прямо в браузере, ставится на главный экран как приложение, работает офлайн. Хостинг — бесплатный GitHub Pages.

📖 **Полная документация:** [DOCS.md](DOCS.md) &nbsp;·&nbsp; 🌐 **Сайт:** https://bablakov.github.io/calculator-vesov/

## Файлы

| Файл | Назначение |
|---|---|
| `index.html` | Всё приложение (разметка + стили + логика в одном файле) |
| `manifest.webmanifest` | Метаданные PWA (имя, иконки, цвета) |
| `sw.js` | Service worker — офлайн-кэш |
| `icon.svg` | Векторная иконка |
| `icon-192/512/180.png` | Растровые иконки (Android + iOS) |
| `tools/gen-icons.mjs` | Генератор PNG-иконок из фигуры (чистый Node) |
| `qr.svg` | QR-код на адрес приложения (для раздачи) |
| `tools/gen-qr.mjs` | Генератор `qr.svg` (dev-зависимость `qrcode`) |

## Что менять под себя

Всё — в начале `<script>` в `index.html`:

```js
const LIFTS = [                                   // упражнения и их цвета
  { key:"squat",    name:"Присед",        accent:"#ff5d3b" },
  { key:"bench",    name:"Жим лёжа",      accent:"#3b82f6" },
  { key:"deadlift", name:"Становая тяга", accent:"#22c55e" },
];
const PERCENTS = [40,50,60,65,70,75,80,85,90,95,100,105]; // ряд процентов
```

## Локальный запуск

Service worker требует http(s) (или localhost). Подними любой статический сервер из папки проекта:

```bash
python -m http.server 8000
# открой http://localhost:8000
```

Сам калькулятор работает и при простом открытии `index.html` двойным кликом — без офлайн-кэша и установки.

## Публикация на GitHub Pages

1. Создай репозиторий и залей в него эти файлы (ветка `main`).
2. **Settings → Pages → Build and deployment → Source: Deploy from a branch**, ветка `main`, папка `/ (root)`.
3. Через минуту приложение доступно по адресу `https://<логин>.github.io/<репозиторий>/`.
4. Поделись ссылкой или QR-кодом. На телефоне: меню браузера → **«Добавить на главный экран»**.

После изменения файлов подними версию кэша в `sw.js` (`pct-v1` → `pct-v2`), иначе у пользователей останется старая версия из кэша.

## Перегенерировать иконки

После смены цвета/фигуры:

```bash
node tools/gen-icons.mjs
```

## Перегенерировать QR

Если сменится адрес сайта:

```bash
npm install     # один раз — поставит dev-зависимость qrcode
npm run qr      # или: node tools/gen-qr.mjs "https://новый-адрес/"
```

## Обновления на устройствах

Обновление прилетает само: после `git push` GitHub Pages пересобирается, а приложение проверяет новую версию при каждом запуске и при возврате к нему. Как только новая версия готова — страница тихо перезагружается; введённые веса и история не теряются (хранятся в `localStorage`). При изменении файлов поднимайте версию `CACHE` в `sw.js`.
